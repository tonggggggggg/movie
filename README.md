# 線上電影票購買系統的設計與實現
**11224137鄭宇彤 11224121黃靖雅**

## 一、需求說明書

報告是以「線上電影票購買系統」為主題，主要目的是讓使用者可以透過網路完成買電影票的流程，不需要特地到現場排隊購票。系統會模擬實際電影院的線上訂票方式，讓使用者可以查看電影資訊、選擇場次與座位，並完成訂票與付款。

在這個專案中，重點放在系統整體流程是否清楚，以及各個功能之間是否能順利配合運作。

使用者可以做的事情：

1. 查看目前上映的電影

2. 選擇想看的電影場次

3. 選擇座位並進行訂票

4. 完成付款（以模擬方式呈現）

5. 查詢自己的訂單紀錄

## 二、概要設計說明書

系統在設計時，會把功能分成幾個主要部分，讓每個功能都有自己的角色，這樣系統在使用或修改時會比較清楚，也比較不容易出錯。

整體來說，系統會分成「使用者操作畫面」以及「後端處理資料」兩大部分，使用者透過畫面進行操作，系統則在背後負責處理電影資料、座位狀態和訂單內容。

系統大致分為：

1. 使用者操作介面

2. 電影與場次資料處理

3. 座位狀態管理

4. 訂單與付款處理

## 三、詳細設計說明書

1. 電影管理

系統會提供電影相關資訊，包含電影名稱、簡單介紹、片長與分級，讓使用者在購票前能先了解電影內容。

2. 場次管理

每部電影都會安排不同的播放時間與影廳，使用者可以依照自己的時間選擇合適的場次。

3. 座位管理

系統會顯示影廳的座位配置，使用者可以直接看到哪些座位已經被選走，避免重複選到同一個座位。

4. 訂單與購票流程

使用者依序選擇電影、場次與座位後，系統會建立一筆訂單，並顯示訂單狀態，讓使用者清楚目前是否完成購票。

5. 付款

系統會模擬付款成功或失敗的情況，並依照結果更新訂單與座位狀態，讓整個購票流程更完整。

6. 系統管理與安全

系統會提供登入功能，區分一般使用者與管理者，確保操作內容不會混亂。

## 四、測試計畫

（確保系統可以正常使用）

在完成系統後，會透過實際操作來測試各項功能是否正常，例如：

1. 是否能順利完成購票流程

2. 座位是否會正確顯示已售或可選

3. 訂單狀態是否會隨付款結果改變

4. 透過這些測試，確認系統在基本使用情況下可以正常運作。
